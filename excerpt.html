Monkey Mind

Too many tabs open — a stochastic duet
Curt Rode

About This Work-in-Progress Excerpt

Monkey Mind is an interactive electronic literature piece in which two voices generate text simultaneously, assembling phrases from overlapping but distinct word pools. Each voice combines relational subjects ("you are my," "I am not your," "we were our") with abstract and pastoral predicates ("silence," "horses," "yellow bird," "darkness"), producing a continuous stream of stochastic language that evokes two minds thinking past each other.

The piece runs in a browser with a terminal-style interface and can be experienced live at curtrode.github.io/monkey_mind. The voices begin muted; the viewer chooses when — and whether — to listen. Text-to-speech narration uses both synthetic machine and AI-generated voices; in the AI mode, the two voices are spatialized in stereo, one left and one right. Enabling speech transforms parallel monologues into something closer to dialogue, making the listener complicit in bridging two minds.

The work is built on Nick Montfort's Memory Slam — a collection of stochastic text engines reimplementing classics like Christopher Strachey's Love Letters (1952) and Alison Knowles & James Tenney's A House of Dust (1967). Where those foundational works operate as poetic monologues, Monkey Mind attempts to carry stochastic generation into dialogue.

What follows is the core generative source of the piece: the generation engine, the word pools that constitute its vocabulary, and a sample of the output it produces. The full project, including interface and speech logic, is available at github.com/curtrode/monkey_mind. In computational writing, the code is the writing — the literary decisions are embedded in the choice of words, the sentence templates, the timing, and the silences.

How to Read This

The Engine (p. 2–3) shows the generation logic — how subjects, predicates, and conjunctions are combined. Note the simplicity: a single function assembles each phrase.

The Word Pools (p. 4–7) are the poem's raw material. Two voices share most vocabulary but diverge in small, deliberate ways. Empty strings in the arrays are intentional — they create gaps, half-sentences, silence.

Sample Output (p. 8–10) captures both voices running simultaneously, as they would appear in the terminal interface.

'use strict';

const MonkeyMind = {
  init(config) {
    const {
      subjects,
      predicates,
      conjunctions,
      operators,
      interval,
      maxVisible = 8,
      endearments = null,
      interjections = null,
      id = null,
    } = config;

    let t = 0;

    const randRange = (maximum) =>
      Math.floor(Math.random() * (maximum + 1));

    const choose = (array) =>
      array[randRange(array.length - 1)];
choose() selects a random element from any array — the single point of stochastic decision-making in the entire engine. 
    const phrase = () => {
      let text = choose(operators);
      if (endearments) {
        text += choose(endearments);
      }
      text += ' ' + choose(subjects);
      if (text === 'A EYE') {
        text = 'AN EYE';
      }
      return text + ' ';
    };
A phrase is: an operator (currently empty) + an optional endearment + a subject. The 'A EYE' → 'AN EYE' correction is a vestige of the original Montfort engine, preserved here. 
    const litany = () => {
      const main = document.getElementById('main');
      if (maxVisible > t) {
        t += 1;
      } else {
        main.removeChild(main.firstChild);
      }
      const last = document.createElement('div');
      let text = phrase() + choose(predicates);
2 
      if (interjections) {
        text += choose(interjections);
      }
      text += choose(conjunctions);
      text += phrase() + choose(predicates) + ' ';
      last.innerHTML = text;
      main.appendChild(last);

      // Send text to parent frame for speech
      if (window.parent !== window && id) {
        window.parent.postMessage(
          { type: 'monkeyMindText', text: text, source: id }, '*'
        );
      }
    };
Each line of output follows the pattern: phrase + predicate + conjunction + phrase + predicate. This is the sentence template — a two-clause structure joined by a conjunction, directly descended from the original stochastic text engines. The postMessage call sends the generated text to the parent frame, where it can be spoken aloud. 
    let currentInterval = interval;
    let timer = null;

    const schedule = () => {
      timer = setTimeout(() => {
        litany();
        schedule();
      }, currentInterval);
    };

    // Listen for interval changes from parent
    window.addEventListener('message', (e) => {
      if (e.data && e.data.type === 'setInterval') {
        currentInterval = e.data.interval;
        clearTimeout(timer);
        schedule();
      }
    });

    litany();
    schedule();
  }
};
Each voice runs on its own timer. "Him" defaults to 3500ms; "her" to 2000ms — she generates nearly twice as fast. The viewer can adjust both speeds. This asymmetry creates the overlapping, unsynchronized rhythm that gives the piece its texture.

The Word Pools

Each voice is initialized with its own arrays of subjects, predicates, and conjunctions. The two voices share most of their vocabulary but diverge in small, deliberate ways — giving each a subtle personality within the shared language of the relationship. Words highlighted in yellow are unique to that voice.

Empty strings appear intentionally throughout the arrays. When selected, they produce gaps — silence, half-formed thoughts, lines that trail into nothing. The ratio of empty to filled entries controls how often each voice falls quiet.

7 of 21 entries are empty — roughly one-third of the time, the subject position is silent. The relational pronouns cycle through tenses and possessives: present ("you are"), past ("she was"), negated ("I am not"). Every subject frames a claim about belonging.

Predicates
[silence] gift silence strength value one meadow dream marrow supply horses breath heart map novelty quiet distance yellow bird cotton darkness freedom expense age anger moth
The predicates mix registers: abstract emotional states (silence, darkness, freedom, anger), pastoral imagery (meadow, horses, yellow bird), and the body (marrow, breath, heart). Map and cotton are unique to "him" — concrete, tactile, navigational.
Conjunctions
[silence] and or but therefore (please listen) (I hear you) (please don't)
The conjunctions include three parenthetical interjections — whispered asides that break the formal sentence pattern. These are the only moments where a voice addresses its listener directly.

Subjects

"Her" voice adds two unique subjects: "I am not" (a bare negation, unfinished — identity dissolving into predicate) and "you are completely" (an intensifier that charges whatever follows). She also has one fewer silence, making her slightly more verbal.
6
Predicates
[silence] [silence] [silence] [silence] [silence] right wrong gift silence strength value one meadow dream marrow supply horses breath heart road novelty quiet distance yellow bird forsythia darkness freedom expense age anger moth
"Her" predicates include 4 more silences — but her faster interval (2000ms vs 3500ms) means she generates more frequently despite the gaps. Her unique words: right and wrong (moral judgment), road (movement, departure), and forsythia (a bright yellow flowering shrub — the most specific, sensory word in the entire piece).
Conjunctions
[silence] and or but therefore (please listen) (I hear you) (please don't)
Identical conjunctions. The parenthetical asides — (please listen), (I hear you), (please don't) — are shared. Both voices have the same capacity for direct address; neither has a monopoly on vulnerability.
